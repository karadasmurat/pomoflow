# Contributing to PomoFlow

Thanks for your interest in contributing to PomoFlow! We want to keep this process simple and productive.

## How to Contribute

### 1. Reporting Issues
*   **Search First:** Before opening a new issue, please check if it has already been reported.
*   **Use Templates:** Use our [Bug Report](https://github.com/karadasmurat/pomoflow/issues/new?template=bug_report.yml) or [Feature Request](https://github.com/karadasmurat/pomoflow/issues/new?template=feature_request.yml) forms to provide the necessary details.

### 2. Proposing Changes
If you'd like to fix a bug or add a feature:
1.  **Open an Issue:** Describe the change you want to make so we can discuss it.
2.  **Branching:** Create a new branch using the naming convention:
    *   `fix/description-of-fix` for bug fixes.
    *   `feat/description-of-feature` for new features.
3.  **Pull Request:** Submit your changes via a Pull Request. Ensure your code follows the existing style and is well-tested.

## Project Vision
PomoFlow is a minimalist productivity tool. We value:
*   Simplicity over complexity.
*   Privacy (everything stays in local storage).
*   Visual consistency and smooth interactions.

Thank you for helping make PomoFlow better!
